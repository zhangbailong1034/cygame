## 1. 产品概述

### 1.1 游戏名称
**成语拼拼乐**

### 1.2 核心玩法
- 界面呈现一个**待填充的成语矩阵**（例如 4×5 或 6×6 网格），部分格子已固定显示汉字，部分格子为**空格**。
- 屏幕下方散落着**汉字/词组碎片**（如"如"、"欢声"、"接风"等）。
- 用户需要将碎片**点击**放入正确的空格中，使**所有横向和纵向的成语都成立**。
- 每个碎片可能包含**一个汉字**（length=1），也可能包含**一个两字词**（length=2，如"欢声"），放入时占据连续的两个格子。
- 支持"提示指定空格"功能：用户点击一个空格后点提示，系统返回正确碎片。

### 1.3 关卡模式

#### 基础模式（1-100关）
- 成语在网格中横向或纵向排列，互不交叉（分布在不同的行/列）。
- 网格尺寸从 2×4 逐步增加到 6×6。
- 部分格子预填固定汉字作为提示，其余格子由玩家从碎片区选取填充。

#### 纵横字谜模式（101-200关）
- 成语在网格中**交叉排列**：横向和纵向成语共享交叉点上的汉字。
- 例如"欢声如雷"横向放置，"如雷贯耳"纵向放置，共享"如""雷""贯"等字。
- 玩法更具挑战性，模拟传统填字游戏的体验。

### 1.4 与普通填字游戏的区别
- 碎片不仅限于单字，可以是**多字词块**，增加拼合乐趣。
- 碎片包含干扰项，需要用户辨别。
- 纵横字谜模式下成语相互交叉，一字多用。

## 2. 用户界面设计

### 2.1 菜单首页
- 顶部显示游戏标题"成语拼拼乐"。
- 以 3 列网格展示关卡按钮，每页 18 个。
- 当前关卡金色高亮，已完成关卡蓝色，未解锁关卡灰色（可点击预览）。
- 底部显示上一页/下一页按钮及页码指示器。
- 最底部显示"排行榜"和"学习记录"按钮。
- 连续点击标题 5 次进入关卡编辑器（隐藏功能）。

### 2.2 游戏主界面

```
+--------------------------------------------------+
| ← 返回      第 2 关 [预览]              ⭐ 100   |
+--------------------------------------------------+
| ❤️❤️❤️ ×3  |  🀫 剩余空格：12 格    完成 45%    |
+--------------------------------------------------+
|     [欢] [声] [□]  [□]  [□]                    |
|     [□]  [□]  [□]  [□]  [□]                    |
|     [□]  [□]  [如] [□]  [□]                    |
|     [□]  [□]  [□]  [贯] [□]                    |
|     [交] [头] [□]  [□]  [□]                    |
+--------------------------------------------------+
|  碎片区：                                        |
|  [接风]  [雷]  [耳]  [动]  [欢声]               |
|  [天] [地]  (干扰项带米色背景)                   |
+--------------------------------------------------+
|        [提示]    [洗牌]    [重置]                |
+--------------------------------------------------+
```

- 固定汉字为绿色渐变背景，空格为白色。
- 碎片区支持多行自动换行，2字碎片宽度加倍。
- 选中碎片橙色高亮边框，选中空格橙色高亮。

### 2.3 预览模式
- 任意关卡均可点击进入查看。
- 非当前顺序关卡的关卡进入后标题显示"[预览]"标识。
- 预览模式下所有操作按钮（提示、洗牌、重置、碎片放置、格子点击）均被禁用，仅可查看。
- 只有当前待闯关的关卡可以进行实际填充操作。

## 3. 关卡数据结构

### 3.1 网格定义
- 最大支持 7×8 网格。
- 每个格子有三种状态：
  - `fixed`：固定汉字，不可编辑，游戏开始时已显示。
  - `empty`：空格，等待用户填充。
  - `filled`：已由用户正确填充。
- 碎片区包含正确碎片和干扰碎片。

### 3.2 关卡配置示例（JSON）

```json
{
  "levelId": 102,
  "rows": 4,
  "cols": 5,
  "fixedCells": [
    {"row": 0, "col": 0, "char": "欢"},
    {"row": 0, "col": 1, "char": "声"}
  ],
  "idioms": [
    {"direction": "horizontal", "row": 0, "startCol": 0, "endCol": 3, "answer": "欢声如雷"},
    {"direction": "vertical", "col": 2, "startRow": 0, "endRow": 3, "answer": "如雷贯耳"}
  ],
  "fragments": [
    {"text": "欢声", "length": 2, "positions": [[0,0],[0,1]]},
    {"text": "雷", "length": 1, "positions": [[0,3]]}
  ],
  "distractors": [
    {"text": "天", "length": 1},
    {"text": "地", "length": 1}
  ]
}
```

## 4. 核心游戏规则

### 4.1 正确性校验
- 碎片放置校验在**服务端**执行，客户端不做预判。
- 放置正确：碎片保留，积分+5，碎片从碎片区移除。
- 放置错误：碎片退回，体力-1，显示"位置不对"。
- 多字碎片需选择连续的两个格子（同行或同列且相邻），否则提示。

### 4.2 通关检测
- 服务端放置碎片时检测是否所有碎片已正确填充。
- 通关后弹出对话框展示所有成语及释义，可选"下一关"或"返回"。
- 通关后自动解锁下一关（`currentLevel` 更新）。

### 4.3 共享格子（纵横字谜）
- 交叉点上的汉字必须同时满足横向和纵向成语的要求。
- 后台配置时保证交叉点唯一且合理。

## 5. 提示系统

- 用户先点击一个空格，再点击"提示"按钮。
- 系统返回该空格应填入的正确碎片文本。
- 提示功能在预览模式下不可用。

## 6. 体力与积分系统

| 行为 | 体力变化 | 积分变化 |
|------|----------|----------|
| 碎片放置错误 | -1 | 0 |
| 正确放置一个碎片 | 0 | +5 |
| 通关奖励 | 0 | +50 |
| 使用提示 | 0 | -10 |
| 洗牌（重排碎片区） | 0 | -5 |
| 重置关卡 | 0（服务端决定） | 0 |

- 体力上限 10，初始 10。

## 7. 关卡难度阶梯

| 难度 | 关卡范围 | 网格尺寸 | 成语数量 | 模式 |
|------|----------|----------|----------|------|
| 简单 | 1-15 | 2×4 | 1-2 | 纯横向 |
| 简单 | 16-30 | 3×4 | 2 | 纯横向 |
| 中等 | 31-50 | 4×4 | 2-3 | 混合横/纵 |

| 纵横-困难 | 171-200 | 6×7 ~ 7×8 | 5-7 | 成语交叉 |

## 8. 后台管理功能

### 8.1 关卡编辑器（内置于游戏中）
- 通过菜单页连续点击标题 5 次进入。
- 可自定义网格尺寸、添加成语、固定字、碎片。
- 支持保存关卡到服务器。

### 8.2 关卡生成器
- 服务端脚本自动生成关卡配置 JSON。
- 支持基础模式（独立行列）和纵横字谜模式（交叉算法）。
- 交叉算法：基于字符索引匹配可交叉成语，自动计算网格尺寸。

## 9. 埋点需求

| 事件名 | 触发时机 | 参数 |
|--------|----------|------|
| fragment_place | 放置一个碎片 | level_id, fragment_text, is_correct |
| hint_use | 使用提示 | level_id, row, col |
| level_reset | 重置关卡 | level_id |
| level_complete | 通关 | level_id, score_delta |

## 10. 开发注意事项

1. **多字碎片的交互判定**：用户点击多字碎片后进入"选择连续空格"模式，必须选择同行或同列相邻的两个空格。
2. **碎片与格子关联**：同一汉字可能出现在多个碎片中，每个空格唯一对应一个正确碎片。
3. **预览模式隔离**：预览模式下不能有任何修改操作，只能查看网格和碎片布局。
4. **分页**：200关需分页展示，每页18个（3列×6行），支持翻页导航。

# 产品需求文档：成语碎片拼合游戏（微信小程序）

| 文档版本 | 修改日期 | 修改人 | 修改内容 |
| :--- | :--- | :--- | :--- |
| V1.0 | 2026-05-09 | AI助手 | 基于碎片拼合+交叉成语玩法初版 |

## 1. 产品概述

### 1.1 游戏名称（示例）
**成语拼拼乐** / **成语碎碎合**

### 1.2 核心玩法
- 界面呈现一个**待填充的成语矩阵**（例如 3×3 或 4×4 网格），部分格子已固定显示汉字，部分格子为**空格**。
- 屏幕下方（或侧边）散落着**汉字/词组碎片**（如“如”、“欢声”、“贯”、“交头”、“接风雷耳动”等）。
- 用户需要将碎片**拖拽或点击**放入正确的空格中，使**所有横向和纵向的成语都成立**。
- 每个碎片可能包含**一个汉字**，也可能包含**一个两字词**（如“欢声”），放入时会占据连续的两个格子。
- 支持“提示指定空格”功能：用户点击某个空格，系统高亮显示正确的碎片（或直接填入）。

### 1.3 与普通填字游戏的区别
- 碎片不仅限于单字，可以是**多字词块**，增加拼合乐趣。
- 碎片可能多于空格（含干扰项），需要用户辨别。

## 2. 用户界面设计

### 2.1 游戏主界面（以您提供的样例为例）

```
+--------------------------------------------------+
| ← 返回            第 2 关               ⭐ 100   |
+--------------------------------------------------+
|     [欢] [声] [□]  [□]  [□]                    |
|     [□]  [□]  [□]  [□]  [□]                    |
|     [□]  [□]  [如] [□]  [□]                    |
|     [□]  [□]  [□]  [贯] [□]                    |
|     [交] [头] [□]  [□]  [□]                    |
+--------------------------------------------------+
|  碎片区（可拖拽/点击）：                         |
|  [接风]  [雷]  [耳]  [动]  [欢声] (已用变灰)    |
|  [贯] (已用)  [如] (已用)  [交头] (已用)        |
|  [多余碎片: 天] [地]                            |
|                                                  |
|  [提示指定空格]  [洗牌碎片]  [重置关卡]          |
+--------------------------------------------------+
|  体力: ❤️❤️❤️  |  剩余空格: 12                   |
+--------------------------------------------------+
```

**说明**：
- 固定汉字（如“欢”“声”“如”“贯”“交”“头”）已经是预先放置好不可编辑的。
- 空格（□）需要用户从下方碎片区选取合适的碎片填入。
- 碎片可以是**单字**（如“雷”“耳”“动”）或**二字词**（如“接风”）。
- 最终要形成的成语示例（推断）：
  - 横向第一行：欢声 + ？+ ？ → “欢声如雷”？但需要 4 字，可能“欢声”占前两格，后两格填入“如”“雷”。
  - 横向第五行：交头 + ？+ ？ → “交头接耳”
  - 纵向某一列：？+ ？+ 如 + 贯 + ？ → 可能“如雷贯耳”（但“如”在第3行第3列，“贯”在第4行第4列？不连续，需要调整布局）

> 实际关卡设计时，后台会定义好网格和成语，确保碎片可以恰好填满所有空格。

### 2.2 碎片交互细节
- **单字碎片**：点击后变为“选中”状态，再点击一个空格，填充该字到空格。
- **多字碎片**（如“接风”）：点击后，需要用户依次点击两个**连续的空格**（横向或纵向），系统将这两个字依次填入。
- 若碎片放置位置错误（不成词或与周围已有的字冲突），碎片自动弹回，并提示“位置不对”。
- 每个碎片只能使用一次，使用后从碎片区移除（或变灰不可再选）。

## 3. 关卡数据结构

### 3.1 网格定义
- 最大支持 6×6 网格。
- 每个格子有三种状态：
  - `fixed`：固定汉字，不可编辑，游戏开始时已显示。
  - `empty`：空格，等待用户填充。
  - `filled`：已由用户正确填充。
- 碎片区包含的碎片由后台根据所有空格的正确答案生成。

### 3.2 关卡配置示例（JSON）

```json
{
  "levelId": 2,
  "rows": 5,
  "cols": 5,
  "fixedCells": [
    {"row": 0, "col": 0, "char": "欢"},
    {"row": 0, "col": 1, "char": "声"},
    {"row": 2, "col": 2, "char": "如"},
    {"row": 3, "col": 3, "char": "贯"},
    {"row": 4, "col": 0, "char": "交"},
    {"row": 4, "col": 1, "char": "头"}
  ],
  "idioms": [
    {"direction": "horizontal", "row": 0, "startCol": 0, "endCol": 3, "answer": "欢声如雷"},
    {"direction": "horizontal", "row": 4, "startCol": 0, "endCol": 3, "answer": "交头接耳"},
    {"direction": "vertical", "col": 2, "startRow": 0, "endRow": 4, "answer": "？"}  // 如雷贯耳？需要2行2列开始？暂略
  ],
  "fragments": [
    {"text": "接风", "length": 2, "positions": [[0,2],[0,3]]},  // 需要填入“接”“风”
    {"text": "雷", "length": 1, "positions": [[0,4]]},
    {"text": "耳", "length": 1, "positions": [[4,2]]},
    {"text": "动", "length": 1, "positions": [[4,3]]}
    // 干扰碎片：“天”“地”
  ]
}
```

- `fragments` 中的 `positions` 指定了该碎片应占用的格子坐标（按顺序）。
- 游戏开始时，系统根据 `fragments` 生成碎片区，并随机打乱顺序。
- 干扰碎片（不在 `fragments` 中但出现在碎片区的额外项）会被标记为 `isDistractor: true`。

## 4. 核心游戏规则

### 4.1 正确性校验
- 每当用户放置一个碎片（或单字），系统立即检查该碎片占据的所有格子上的字是否与对应位置的**正确答案**匹配（正确答案由后台 `idioms` 推导得出）。
- 若**所有格子的字都匹配**，则保留填充，碎片从碎片区移除；若**任何一个格子不匹配**，整个碎片被退回，并扣除 1 点体力。
- 如果放置的是一个多字碎片，但用户点击的两个空格不连续或方向不一致（比如一个横向一个纵向），则提示“请选择连续的两个格子（同一行或同一列）”，碎片退回。

### 4.2 自动完成检测
- 当所有空格都被正确填充后，自动通关。
- 播放通关动画，并展示每个成语的释义。

### 4.3 特殊处理：同一个汉字出现在多个成语中的共享格子
- 如果一个格子同时属于横向成语和纵向成语，其汉字必须同时满足两个成语的要求。后台配置时需保证交叉点唯一且合理。

## 5. 提示系统（符合“可提示指定空格”）

### 5.1 提示指定空格
- 用户**点击一个空格**，再点击“提示”按钮（或长按空格弹出提示选项）。
- 系统自动判断该空格应该填入哪个碎片中的哪个汉字，并做以下之一的操作（可配置）：
  - **高亮正确的碎片**：在碎片区闪烁显示拥有该字的碎片。
  - **直接填入正确汉字**：将正确的字填入该空格（如果该字属于一个多字碎片，则仅填入当前格子，但会打乱碎片的完整性？需要设计规则）。
- 推荐**只高亮碎片**，不自动填充，保留游戏挑战性。
- 每次提示消耗 10 积分或观看激励视频。

### 5.2 其他提示
- **显示成语解释**：点击“成语词典”按钮，查看当前关卡所有成语的释义（消耗 5 积分）。
- **揭示一个完整成语**：观看视频后，系统自动填满一个成语的所有空格（强烈提示，每关限用 1 次）。

## 6. 体力与积分系统

| 行为 | 体力变化 | 积分变化 |
|------|----------|----------|
| 开始新关卡 | 无 | 无 |
| 碎片放置错误（与正确位置不符） | -1 | 0 |
| 正确放置一个碎片 | 0 | +5（每个碎片） |
| 通关 | 奖励 +2（不超过上限） | +50 |
| 使用“高亮提示” | 0 | -10 或 看视频 |
| 使用“洗牌”（重排碎片区） | 0 | -5 |
| 重置关卡 | -1（或免费一次） | 0 |

## 7. 关卡难度阶梯

| 难度 | 网格尺寸 | 成语数量 | 碎片总数（含干扰） | 多字碎片占比 |
|------|----------|----------|-------------------|--------------|
| 简单 | 3×3 | 2 个 | 6~8 | 低（0~1个） |
| 中等 | 4×4 ~ 5×5 | 3~4 个 | 10~14 | 中（2~3个） |
| 困难 | 5×5 ~ 6×6 | 5~6 个 | 15~20 | 高（4~6个） |

## 8. 后台管理功能

### 8.1 关卡编辑器
- 可视化绘制网格，设置固定汉字。
- 定义横向和纵向成语，系统自动计算所有空格的正确答案。
- 根据正确答案**自动生成碎片列表**（将连续的空格合并为多字碎片，也可手动拆分）。
- 添加干扰碎片（随机从常用字库抽取不属于正确答案的汉字）。
- 预览游戏效果。

### 8.2 数据统计
- 每关的通过率、平均耗时、提示使用次数。
- 用于调整关卡难度。

## 9. 埋点需求

| 事件名 | 触发时机 | 参数 |
|--------|----------|------|
| fragment_place | 放置一个碎片 | level_id, fragment_text, is_correct, remaining_fragments |
| hint_space | 对指定空格使用提示 | level_id, row, col, hint_type(highlight/fill) |
| level_reset | 重置关卡 | level_id |

## 10. 开发注意事项

1. **多字碎片的交互判定**：用户点击多字碎片后，需要进入“选择连续空格”模式。界面上需提示“请选择两个连续的空格”。必须确保用户选择的方向与后台预设的方向一致（例如后台要求横向，用户选了纵向则报错）。
2. **碎片与格子关联**：同一个汉字可能出现在多个碎片中（例如“雷”可能单独是一个碎片，也可能包含在“雷动”碎片中）。后台需确保每个空格唯一对应一个正确的碎片。
3. **性能**：每次放置碎片后，需重新计算剩余空格数并检查是否通关。
